# CLCOMX 0.1.3

릴리즈 성격: 패치 릴리즈

## 변경점

- Windows 빌드 경로를 보강했습니다.
  - WSL에서 Windows `cargo`/`node` 탐지를 더 견고하게 정리
  - 환경변수 override 안내 추가
  - 설치본과 포터블 zip을 함께 빌드하도록 정리
- Windows 에디터 감지를 보강했습니다.
  - PATH, known path 외에 `App Paths` registry fallback 추가
  - VS Code, Cursor, Windsurf, Sublime Text, Notepad++, PhpStorm 감지 안정화
- GitHub Actions를 보강했습니다.
  - Node 24 대응
  - 릴리즈 workflow에서 프론트 `dist` 빌드 후 Tauri 빌드
  - installer와 portable zip 업로드 경로 정리
- 보조 터미널을 안정화했습니다.
  - hide/show 렌더 밀림 문제 수정
  - 실제 셸 cwd를 따라 헤더 경로가 갱신되도록 개선
  - 웹뷰 새로고침 시에는 기존 보조 터미널 PTY에 다시 attach 하도록 보강
  - 앱 종료와 탭 종료 시에는 보조 터미널을 정리하는 정책 유지
- 웹뷰 새로고침 시 메인 에이전트 터미널도 기존 PTY에 다시 attach 하도록 보강했습니다.

## 검증

- `npm run test`
- `npm run check:frontend`
- `cargo test --manifest-path src-tauri/Cargo.toml`
- `bash scripts/build-dev.sh`
- `bash scripts/build-windows.sh`

## 비고

- `저장소에 올려`는 커밋/푸시만 수행하고 태그는 만들지 않습니다.
- `저장소에 올리고 배포` 지시가 있을 때만 태그를 만들고 release workflow를 실행합니다.
