# CLCOMX 0.2.0

## Added

- Runtime-extensible terminal themes via execution-directory `theme.json`
  - Built-in theme pack moved to JSON
  - `theme.json` is created automatically on first run
  - User themes can override built-ins by reusing the same `id`
  - `extends` allows partial theme customization
- Main terminal loading overlay with animated attach/restore states
- Background editor detection cache shared by settings and terminal file-open flows

## Improved

- Windows editor detection now uses:
  - override env vars
  - PATH / `where.exe`
  - known install paths
  - registry `App Paths`
  - common install roots
- Windows build flow now produces both:
  - NSIS installer
  - portable zip
- Release workflow now builds frontend assets before Tauri packaging and uploads the portable zip artifact too

## Fixed

- Settings modal no longer stalls while synchronously detecting editors
- Main terminal loading overlay now stays visible until rendered output stabilizes
- Auxiliary terminal hide/show rendering corruption fixed
- Auxiliary terminal header path now tracks the real shell cwd
- Webview refresh now reattaches both main and auxiliary PTYs without losing live session state
