# CLCOMX 0.3.1

릴리즈 성격: 패치 릴리즈

## 변경점

- Claude footer ghosting 완화를 기본 동작으로 올렸습니다.
  - 배포 빌드에서도 기본적으로 완화가 켜집니다.
  - 기존 실험 플래그 의존 때문에 재현되던 배포본 회귀를 줄였습니다.
- 터미널 설정에 Claude 전용 옵션을 추가했습니다.
  - 설정에서 `Claude footer ghosting 완화`를 켜고 끌 수 있습니다.
  - 현재 환경에서 회귀가 있을 때만 끌 수 있도록 기본값은 켜짐입니다.
- 내부 적용 경로를 정리했습니다.
  - 메인 터미널, 보조 터미널, canonical screen authority가 같은 완화 기준을 사용합니다.
  - `CLCOMX_SOFT_FOLLOW_EXPERIMENT`는 진단용 override로 유지되며, `1`은 강제 켜기, `0`은 강제 끄기입니다.
- GitHub 릴리즈 워크플로를 안정화했습니다.
  - deprecated Node 20 action 의존을 제거했습니다.
  - 태그 릴리즈 시 `gh release` 기반으로 자산 업로드를 수행합니다.

## 검증

- `npm run test`
- `npm run check:frontend`
- `cargo test --manifest-path src-tauri/Cargo.toml`
- `bash scripts/build-dev.sh`

## 비고

- 이번 단계는 버전 반영, 저장소 반영, 태그 배포까지를 포함합니다.
